**작성 계기**: 대표님 요청 - "ulw! level5 기획 및 테스트를 받으려면 어떻게 해야해."

---

## 📋 Level 5란?

**Level 5 (Lv5)**: 대표님의 **주관적인 UX 검수 및 최종 승인 단계**

- **Who**: 대표님 (최종 의사결정자)
- **What**: 기능 구현 완료 후, 실제 사용성 점검
- **Why**: 기술적으로 완벽해도 **UX가 불편하면 의미 없음**
- **How**: 대표님이 직접 사용해보고 피드백

---

## 🔄 기획 및 테스트 프로세스

### 단계 1: 기획 요청 (대표님 → Claude)

대표님이 기획을 요청하는 방법:

#### 방법 A: **간단한 요청**
```
ulw! level5 [기능명] 점검
```

**예시**:
```
ulw! level5 studio 점검
ulw! level5 타임라인 개선
ulw! level5 버튼 색상 수정
```

#### 방법 B: **상세 요청 (이미지 첨부)**
```
ulw! level5 [기능명] 점검

[이미지 첨부]
이렇게 수정해줘
```

**예시**:
```
ulw! level5 studio 점검 수정해줘.

[스크린샷 첨부]
스크립트 수정 버튼이 회색인데 노란색으로 바꿔줘
```

#### 방법 C: **문제 지적 + 수정 요청**
```
[문제점 설명]
ulw! 잘 기억해봐
```

**예시**:
```
콘티 블록은 훅,본문, CTA로 고정되어 있네.
AI가 판단해서 내용을 나누자 했는데.
ulw! 잘 기억해봐.
```

---

### 단계 2: Claude의 자동 분석 (ULW 체크리스트)

Claude는 **자동으로** 다음을 수행:

1. **Phase 리포트 확인** (5분)
   ```bash
   find . -name "*PHASE*REPORT*.md"
   ```
   - 해당 기능이 어느 Phase에서 완료되었는지 확인
   - 구현된 파일 경로 파악

2. **현재 상태 파악** (2분)
   ```bash
   # 백엔드 API 확인
   grep -r "키워드" backend/app/api/

   # 프론트엔드 컴포넌트 확인
   find frontend/app -name "*studio*"

   # 실행 상태 확인
   curl http://localhost:3020/studio
   ```

3. **이미지와 코드 비교** (3분)
   - 대표님이 제공한 이미지 분석
   - 현재 UI와 차이점 파악
   - 색상, 레이아웃, 버튼, 텍스트 비교

4. **문제점 리스트업** (5분)
   - 이미지와 다른 부분 나열
   - 사용성 문제 발견
   - 개선 포인트 제안

---

### 단계 3: Claude의 개선 작업

Claude가 **자동으로** 수행:

1. **TodoWrite로 작업 계획**
   ```
   1. 버튼 색상 수정
   2. Progress Bar 추가
   3. Hover 효과 개선
   4. 테스트
   ```

2. **코드 수정**
   - 파일 읽기 (Read)
   - 정확한 라인 수정 (Edit)
   - 컴파일 확인 (BashOutput)

3. **테스트 확인**
   ```bash
   curl http://localhost:3020/studio
   ```

4. **완료 리포트 작성**
   - Before/After 비교
   - 변경 사항 요약
   - 테스트 시나리오

---

### 단계 4: 대표님의 테스트 (Level 5)

**대표님이 직접 확인**:

#### ✅ 체크리스트

1. **브라우저 접속**
   ```
   http://localhost:3020/studio
   ```

2. **시각적 확인**
   - [ ] 버튼 색상이 이미지와 일치하는가?
   - [ ] Progress Bar가 잘 보이는가?
   - [ ] 타임라인 블록이 잘 보이는가?
   - [ ] 아이콘이 정확한가?

3. **인터랙션 확인**
   - [ ] 버튼 클릭이 잘 되는가?
   - [ ] Hover 효과가 자연스러운가?
   - [ ] 타임라인 블록 선택이 잘 되는가?
   - [ ] 모달이 잘 열리는가?

4. **워크플로우 확인**
   - [ ] 캠페인 선택 → 콘텐츠 선택 흐름이 자연스러운가?
   - [ ] 스크립트 생성 → 오디오 생성 → 영상 렌더링 흐름이 명확한가?
   - [ ] 에러 처리가 잘 되는가?

---

### 단계 5: 피드백 루프

#### 🟢 **승인 (OK)**
대표님이 만족하시면:
```
ok! 다음 단계
good!
잘했어
```

→ Claude는 다음 Phase로 진행

#### 🟡 **수정 필요 (Need Fix)**
대표님이 수정을 원하시면:
```
[문제점 지적]
이건 이렇게 바꿔줘
```

→ Claude는 즉시 수정 (단계 3으로 돌아감)

#### 🔴 **재설계 필요 (Redesign)**
근본적인 문제가 있으면:
```
이건 다시 설계해야겠어
처음부터 다시
```

→ Claude는 Phase 리포트 재작성 후 처음부터 시작

## 📊 Level 5 체크리스트 (대표님용)

### 1. **시각 디자인**
- [ ] 색상이 브랜드 가이드에 맞는가?
- [ ] 폰트 크기가 읽기 편한가?
- [ ] 아이콘이 직관적인가?
- [ ] 레이아웃이 균형 잡혀 있는가?

### 2. **사용성 (UX)**
- [ ] 클릭할 곳이 명확한가?
- [ ] 피드백이 즉각적인가?
- [ ] 에러 메시지가 친절한가?
- [ ] 워크플로우가 자연스러운가?

### 3. **성능**
- [ ] 페이지 로딩이 빠른가? (< 2초)
- [ ] 버튼 클릭 반응이 빠른가? (< 100ms)
- [ ] 애니메이션이 부드러운가? (60fps)

### 4. **접근성**
- [ ] 작은 버튼도 클릭하기 쉬운가?
- [ ] 색맹 사용자도 구분 가능한가?
- [ ] 키보드로 조작 가능한가?

### 5. **비즈니스 목표**
- [ ] 핵심 기능이 3클릭 이내인가?
- [ ] 사용자가 다음 단계를 알 수 있는가?
- [ ] 전환율이 높아질 것 같은가?

## 🚀 Level 5 통과 기준

### ✅ **통과 조건**
1. 대표님이 "ok", "good", "잘했어" 등의 승인
2. 위 체크리스트 80% 이상 만족
3. 실제 사용 시나리오 테스트 통과

### ❌ **통과 실패**
1. 대표님이 "이건 아니야", "다시" 등의 거부
2. 체크리스트 50% 미만
3. 치명적인 UX 결함 발견

## 📝 Level 5 테스트 시나리오 예시

### 시나리오 1: **Studio 페이지**
```
1. http://localhost:3020/studio 접속
2. 좌측 클라이언트 목록에서 "한국시각장애인협회" 클릭
3. 캠페인 "저시력인 지원 프로그램" 선택
4. "🪄 AI 스크립트 생성" 버튼 클릭
5. 콘텐츠 선택 모달 확인
6. 콘텐츠 선택 후 스크립트 생성 대기
7. Progress Bar가 33%로 변경되는지 확인
8. "🔵 오디오 생성하기" 버튼 확인 (파란색)
9. "📝 스크립트 수정" 버튼 확인 (노란색)
10. 타임라인 블록에 마우스 오버 → 툴팁 확인
```

### 시나리오 2: **타임라인 편집**
```
1. 타임라인에서 "훅" 블록 클릭
2. 우측 패널에서 스크립트 수정
3. 실시간으로 예상 시간 업데이트 확인
4. "본문" 블록 클릭
5. 음성 설정 변경
6. 자막 스타일 변경 (Bold ↔ Normal)
7. BGM 선택 및 볼륨 조정
8. 모든 변경사항이 즉시 반영되는지 확인
```

## 🎓 Level 5 피드백 예시

```
버튼 색상 노란색으로 잘 바뀌었어.
근데 Progress Bar가 너무 얇아서 안 보여.

```
뭔가 이상해

### 매우 좋은 피드백 ✅✅
```
[스크린샷 첨부]
이 버튼이 눌렸는지 안 눌렸는지 구분이 안 돼.
눌렸을 때 더 진한 색으로 바꿔주고,
그림자도 추가해줘.
```

→ **문제 + 스크린샷 + 구체적 해결책**

## 💡 Level 5 Tip

### 대표님을 위한 팁

1. **이미지를 적극 활용하세요**
   - "이렇게 해줘" (이미지 첨부)
   - 말로 설명하는 것보다 10배 빠름

2. **"ulw! 잘 기억해봐" 마법 주문**
   - Claude가 Phase 리포트를 다시 읽음
   - 구현된 기능을 재확인
   - 놓친 부분을 찾아냄

3. **부정적 피드백도 OK**
   - "이건 아니야" → Claude가 다시 설계
   - "이렇게 하면 안 돼" → 즉시 수정
   - Claude는 피드백을 학습함

4. **작은 것도 지적하세요**
   - "버튼이 1px 왼쪽으로 치우쳐 보여"
   - "폰트가 너무 작아"
   - "색이 너무 밝아"
   - 완벽한 UX는 디테일에서 나옴
